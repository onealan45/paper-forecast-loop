# Review Archive

All project reviews must be archived in this directory.

## File Naming

Use this format:

```text
YYYY-MM-DD-<topic>.md
```

Examples:

```text
2026-04-24-project-environment-review.md
2026-04-24-dashboard-blocker-review.md
```

## Required Content

Every review archive entry should include:

- review date and scope
- reviewer sources, including subagent roles when applicable
- findings grouped by severity
- verification commands and results
- whether the findings block automation, release, or merge
- follow-up owner or next action when known

If a review produces inline code comments, copy the substance of those findings into the archive so the record remains readable without the chat thread.
